2快速部署（5 步上线）

  1. 拉取代码git clone <repo> /opt/cooking-plan && cd /opt/cooking-plan
  2. 配置环境变量cp .env.example .env && vim .env （填入 DB_PASSWORD/JWT_SECRET/ADMIN_PASSWORD 等）
  3. 赋权并启动chmod +x auto_run.sh && ./auto_run.sh start
  4. 等待自动初始化容器启动会自动迁移 schema + 创建管理员（约 30 秒）
  5. 访问验证浏览器打开 http://服务器IP:3000，用 ADMIN_EMAIL/ADMIN_PASSWORD 登录

生成强密钥

# 生成 DB_PASSWORD 和 JWT_SECRET
openssl rand -base64 32
openssl rand -base64 32

⚠️ 生产环境必须修改默认密码！ADMIN_PASSWORD 至少 8 位，建议含大小写字母+数字+符号。

3auto_run.sh 脚本

统一管理脚本，自动使用 docker-compose.prod.yml 生产配置：

命令功能
./auto_run.sh start启动服务（构建镜像 + 启动容器）
./auto_run.sh stop停止服务（保留数据卷）
./auto_run.sh restart重启服务
./auto_run.sh status查看服务运行状态 + 资源占用
./auto_run.sh logs [服务名]查看日志（app / db）
./auto_run.sh updategit pull + 重新构建 + 重启
./auto_run.sh backup备份 Postgres（gzip 压缩，自动清理 30 天前）
./auto_run.sh restore <file>从备份恢复（停服务 → 恢复 → 启服务）
./auto_run.sh help显示帮助

脚本内部使用 docker compose -f docker-compose.prod.yml（新版 V2 语法）。

4Docker 架构

文件清单

文件用途
Dockerfile多阶段构建（deps → builder → runner）

容器架构

┌─ docker-compose.prod.yml ─────────────────────┐
│                                              │
│  ┌─ app (Next.js 生产镜像) ──────────┐        │
│  │  端口：3000 → 暴露到主机          │        │
│  │  依赖：db 健康检查通过            │        │
│  │  entrypoint.sh：                  │        │
│  │    1. 等 DB 就绪                  │        │
│  │    2. drizzle-kit push（迁移）    │        │
│  │    3. tsx scripts/seed.ts（管理员）│       │
│  │    4. node server.js（启动）      │        │
│  └──────────┬───────────────────────┘        │
│             │                                │
│  ┌─ db (postgres:15-alpine) ────────┐        │
│  │  仅 docker 网络可见（不暴露）     │        │
│  │  数据卷：postgres_data           │        │
│  │  健康检查：pg_isready             │        │
│  └──────────────────────────────────┘        │
│                                              │
│  卷：postgres_data, uploads_data             │
│  网络：cooking_network (bridge)              │
└──────────────────────────────────────────────┘

安全性：Postgres 不暴露端口到主机，仅在 docker 内部网络可见，避免外部攻击。

5环境变量

创建 .env 文件（参考 .env.example）：

# ============ 数据库配置 ============
DB_HOST=db                    # 容器内固定为 db（docker 服务名）
DB_PORT=5432
DB_NAME=cooking_plan
DB_USER=cooking_user
DB_PASSWORD=openssl rand -base64 32

# ============ JWT 会话密钥 ============
JWT_SECRET=openssl rand -base64 32

# ============ 应用配置 ============
NEXT_PUBLIC_BASE_URL=https://your-domain.com

# ============ 管理员配置 ============
ADMIN_EMAIL=admin@your-domain.com
ADMIN_PASSWORD=强密码（至少 8 位）

# ============ 可选：端口配置 ============
APP_PORT=3000                 # 暴露到主机的端口

⚠️ 所有带 openssl rand 标记的项必须用强随机值！

各变量说明

变量必填说明
DB_PASSWORD✅数据库密码（强随机）
JWT_SECRET✅会话签名密钥（≥16 字符）
ADMIN_EMAIL✅管理员邮箱（登录用）
ADMIN_PASSWORD✅管理员密码（首次初始化，可后续改）
NEXT_PUBLIC_BASE_URL✅对外访问 URL
APP_PORT可选默认 3000

修改管理员密码：编辑 .env 中的 ADMIN_PASSWORD，运行 ./auto_run.sh restart 容器会自动更新。

6配置域名访问（推荐）

部署完成后默认用 http://服务器IP:3000 访问。配置域名后可用 https://your-domain.com 访问，更专业更安全。

前置检查

  • 已购买域名（腾讯云/阿里云/Cloudflare 等）
  • 已知道服务器公网 IP（腾讯云控制台查看）
  • 腾讯云安全组已开放 80 和 443 端口

⚠️ 关于备案：中国大陆服务器必须完成 ICP 备案才能用域名访问 80/443 端口。备案需 7-20 天，建议先用 IP:3000 测试。港澳台/海外服务器不需要备案。

步骤 1：DNS 解析（域名 → 服务器 IP）

登录你购买域名的服务商（以 DNSPod/腾讯云为例）：

# DNSPod 控制台 → 域名解析 → 添加记录

主机记录：cook            # 子域名，如 cook.your-domain.com
          或 @             # 直接用主域名 your-domain.com
记录类型：A
记录值：43.135.100.50      # 你的服务器公网 IP
TTL：600                  # 默认即可

添加后等 1-10 分钟生效。验证：ping cook.your-domain.com 应该返回你的服务器 IP。

步骤 2：开放腾讯云安全组端口

# 腾讯云控制台 → 你的实例 → 防火墙/安全组 → 添加规则

协议端口：TCP:80
来源：0.0.0.0/0

协议端口：TCP:443
来源：0.0.0.0/0

（3000 端口可关闭，因为通过 Nginx 转发）

步骤 3：安装 Nginx + Certbot

# 在服务器上执行
sudo apt update
sudo apt install -y nginx certbot python3-certbot-nginx

步骤 4：配置 Nginx 反向代理

# 创建配置文件
sudo vim /etc/nginx/conf.d/cooking-plan.conf

# 写入以下内容（替换 your-domain.com 为你的实际域名）
server {
    listen 80;
    server_name cook.your-domain.com;     # ← 改成你的域名

    # 上传大小限制（匹配应用 5MB）
    client_max_body_size 6m;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
    }
}
# 测试配置语法
sudo nginx -t

# 重载配置生效
sudo systemctl reload nginx

此时访问 http://cook.your-domain.com 应该能看到应用（HTTP 模式）。

步骤 5：申请 SSL 证书（启用 HTTPS）

# 自动申请并配置 Let's Encrypt 免费证书
sudo certbot --nginx -d cook.your-domain.com

# 按提示操作：
# 1. 输入邮箱（用于证书到期提醒）
# 2. 同意服务条款：Y
# 3. 接收邮件：N（可选）
# 4. certbot 会自动修改 Nginx 配置添加 HTTPS

自动续期：certbot 会安装 systemd timer，证书到期前 30 天自动续期。测试续期：sudo certbot renew --dry-run

步骤 6：更新应用配置

# 编辑 .env，把 BASE_URL 改为 HTTPS 域名
vim /opt/cooking-plan/.env

# 修改这一行：
NEXT_PUBLIC_BASE_URL=https://cook.your-domain.com

# 重启应用生效
./auto_run.sh restart

验证清单

  • https://cook.your-domain.com 能正常打开
  • 浏览器地址栏显示🔒锁图标（HTTPS 生效）
  • 注册/登录功能正常（Cookie secure 生效）
  • 图片上传正常
  • 手机访问正常

7运维操作

查看日志

# 实时查看所有日志
./auto_run.sh logs

# 只看应用日志
./auto_run.sh logs app

# 只看数据库日志
./auto_run.sh logs db

数据备份（手动）

./auto_run.sh backup
# 备份文件保存在 ./backups/cooking_plan_YYYYMMDD_HHMMSS.sql.gz
# 自动清理 30 天前的旧备份

定时备份（推荐）

# 编辑 crontab
crontab -e

# 每天凌晨 3 点自动备份
0 3 * * * cd /opt/cooking-plan && ./auto_run.sh backup >> backups/cron.log 2>&1

数据恢复

# 从指定备份恢复
./auto_run.sh restore backups/cooking_plan_20260715_030000.sql.gz

更新部署

# 拉取最新代码 + 重新构建 + 重启
./auto_run.sh update

清理旧镜像

docker image prune -a -f

8常见问题

Q1: 端口 3000 被占用

解决：在 .env 中添加 APP_PORT=8080（或其他端口），重启服务。

Q2: 容器启动后访问不了

# 查看启动日志，定位错误
./auto_run.sh logs app

# 常见原因：
# 1. .env 中 DB_PASSWORD / JWT_SECRET / ADMIN_PASSWORD 未填
# 2. 数据库健康检查未通过（增加 retries）
# 3. sharp 安装失败（检查 alpine 兼容性）

Q3: 数据库密码改了无法连接

原因：Postgres 数据卷已用旧密码初始化，改 .env 不会同步。

解决 A（保留数据）：进容器手动改密码

docker exec -it cooking_plan_db psql -U cooking_user -d cooking_plan -c "ALTER USER cooking_user PASSWORD 'new_password';"

解决 B（丢弃数据）：

./auto_run.sh stop
docker volume rm cooking_plan_postgres_data
# 修改 .env 后重启
./auto_run.sh start

Q4: 忘记管理员密码

解决：修改 .env 中的 ADMIN_PASSWORD，运行 ./auto_run.sh restart， 容器启动会自动用新密码更新管理员账号（幂等）。

Q5: 图片上传后丢失

原因：uploads_data 卷未正确挂载。

检查：docker volume ls | grep uploads

Q6: 内存不足（OOM）

解决：在 docker-compose.prod.yml 的 db 服务加内存限制：

services:
  db:
    ...
    deploy:
      resources:
        limits:
          memory: 512M
