部署架构：Docker Compose 管理 Next.js 应用 + Postgres 数据库， 通过 auto_run.sh 脚本统一管理启动/停止/状态查询。

目录： ① 前置要求 ② 部署步骤 ③ auto_run.sh 脚本 ④ Docker Compose 配置 ⑤ 环境变量 ⑥ 运维操作 ⑦ 常见问题

1前置要求

服务器配置

项目最低配置推荐配置

  • Git — 拉取代码

腾讯云轻量应用服务器可选择「Docker」镜像，预装 Docker 和 Docker Compose。

2部署步骤

  1. 服务器环境准备安装 Docker + Docker Compose + Git（如未安装）
  2. 拉取代码git clone 仓库到服务器 /opt/cooking-plan
  3. 配置环境变量复制 .env.example 并修改配置值
  4. 构建镜像首次运行自动构建 Next.js 生产镜像
  5. 启动服务./auto_run.sh start
  6. 配置域名解析到服务器 IP（可选，推荐）
  7. 配置 SSLNginx 反向代理 + Let's Encrypt（可选）

3auto_run.sh 脚本

统一管理脚本，放置在项目根目录，提供以下命令：

命令功能
./auto_run.sh start启动服务（构建镜像 + 启动容器）
./auto_run.sh stop停止服务（保留数据卷）
./auto_run.sh restart重启服务
./auto_run.sh status查看服务运行状态
./auto_run.sh logs查看服务日志
./auto_run.sh update更新代码并重新部署
./auto_run.sh backup备份 Postgres 数据
./auto_run.sh help显示帮助信息

脚本自动处理 Docker 镜像构建、容器启动、数据卷挂载等细节。

4Docker Compose 配置

docker-compose.yml 定义两个服务：

# 简化示意，实际配置见项目文件
services:
  db:
    image: postgres:15-alpine
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      POSTGRES_DB: cooking_plan
      POSTGRES_USER: ${DB_USER}
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    restart: always

  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      DATABASE_URL: postgresql://${DB_USER}:${DB_PASSWORD}@db:5432/cooking_plan
      JWT_SECRET: ${JWT_SECRET}
      NEXT_PUBLIC_BASE_URL: ${NEXT_PUBLIC_BASE_URL}
    depends_on:
      - db
    restart: always

volumes:
  postgres_data:

数据持久化：Postgres 数据存储在 Docker Volume postgres_data 中， 容器重建不会丢失数据。图片上传使用 Docker Volume uploads。

5环境变量

创建 .env 文件（参考 .env.example）：

# 数据库配置
DB_USER=cooking_user
DB_PASSWORD=强密码（32位随机）

# JWT 密钥（用于会话加密）
JWT_SECRET=强密钥（32位随机）

# 应用配置
NEXT_PUBLIC_BASE_URL=https://your-domain.com

# 上传配置
MAX_UPLOAD_SIZE=5242880
ALLOWED_IMAGE_TYPES=image/jpeg,image/png,image/webp

# 管理员配置（首次注册自动设为管理员）
ADMIN_EMAIL=your@email.com

⚠️ 生产环境必须使用强密码！推荐使用 openssl rand -base64 32 生成。

6运维操作

日志查看

# 查看所有服务日志
docker-compose logs -f

# 只看应用日志
docker-compose logs -f app

# 只看数据库日志
docker-compose logs -f db

数据备份

# 手动备份 Postgres
./auto_run.sh backup

# 备份文件保存在 ./backups/ 目录

数据恢复

# 从备份恢复（停服务 → 恢复 → 启服务）
docker-compose down
docker run --rm -v cooking_plan_postgres_data:/data -v $(pwd)/backups:/backup \
  postgres:15-alpine sh -c "cat /backup/backup.sql | psql -U cooking_user cooking_plan"
docker-compose up -d

更新部署

# 拉取最新代码并重新部署
git pull
./auto_run.sh update

清理旧镜像

docker image prune -a -f

7常见问题

Q1: 端口冲突

问题：3000 端口被占用

解决：修改 docker-compose.yml 中 ports: "3000:3000" 为 "8080:3000"

Q2: 数据库连接失败

问题：app 无法连接 db

解决：检查 depends_on 配置，等待 db 完全启动（健康检查）

Q3: 图片上传失败

问题：上传后 404 或丢失

解决：确保 uploads volume 正确挂载，检查容器内文件权限

Q4: 内存不足

问题：服务器内存吃紧

解决：给 Postgres 限制内存，或在 docker-compose.yml 中添加 mem_limit

Q5: 如何配置 HTTPS

方案一：使用腾讯云负载均衡 + SSL 证书

方案二：服务器上装 Nginx 反向代理 + Let's Encrypt certbot

推荐 Nginx 方案，灵活且免费：

# 安装 certbot
sudo apt install certbot python3-certbot-nginx

# 自动配置 HTTPS
sudo certbot --nginx -d your-domain.com
